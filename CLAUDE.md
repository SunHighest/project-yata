# CLAUDE.md — Project YATA 行動指針

Claude Code がこのプロジェクトで作業する際の最優先ルール集。
PROJECT.md のビジョン・哲学を前提とし、実装判断はここで補完する。

---

## プロジェクト概要

Project YATA の公開コーポレートサイト（静的サイト）。
さくらインターネット レンタルサーバーにホスティング。

---

## 技術スタック

| 項目 | 内容 |
|---|---|
| Framework | Next.js 16 (App Router) |
| 出力形式 | Static Export (`output: "export"`, `trailingSlash: true`) |
| UI | React 19 + Tailwind CSS v4 |
| Animation | Framer Motion |
| Language | TypeScript |
| Hosting | さくらインターネット レンタルサーバー |

---

## ビルド

```bash
npm run build
```

- `.env.production` に `NEXT_PUBLIC_SITE_URL` が必要
- 出力先: `out/` フォルダ
- チャンクファイル名はビルドごとにハッシュが変わる（例: `2__dsaa22zdre.js`）

---

## デプロイ（手動）

**現在は手動デプロイのみ。GitHub Actions 自動デプロイは未稼働。**

手順:
1. `npm run build` を実行
2. FTPクライアント（FileZilla等）でさくらサーバーへ接続（SFTP/ポート22）
3. サーバー上の `_next/` フォルダを**先に削除**する（必須）
4. `out/` の中身を全てアップロードする

> `_next/` を削除せずに上書きすると、旧ビルドの古いチャンクと新ビルドの HTML が混在して黒画面になる。

自動デプロイの課題は [docs/lessons.md](docs/lessons.md) を参照。

---

## さくらインターネット 接続制約（重要）

| 方式 | 結果 | 理由 |
|---|---|---|
| FTP (ポート21) | **NG** | GitHub Actions の IP を遮断 |
| FTPS (ポート21, TLS) | **NG** | 同上 |
| SFTP + パスワード認証 | **NG** | さくらは SSH パスワード認証を禁止 |
| SFTP + 公開鍵認証 | **未設定** | 鍵の登録が必要 |
| 手動アップロード | **OK** | 現在の運用方法 |

## 実装ルール

### やること
- 指示された変更のみを行う。範囲を超えた改善・リファクタリングは行わない
- UI 変更は既存のデザイン・スペーシング・タイポグラフィを踏襲する
- コメントは「なぜ」が自明でない場合のみ記述する

### やらないこと
- 指示なしのアニメーション追加・変更
- 指示なしの抽象化・コンポーネント分割
- 動作確認なしでの「完了」報告
- 失敗が続く手法を繰り返す（3回失敗したら方針を変える）

---

## コミュニケーションルール

- 原因不明の場合は「わからない」と正直に伝える
- エラーが発生したら、原因・影響範囲・修正方針を明確に報告する
- ユーザーへの確認が必要な場合は、選択肢を提示して判断を仰ぐ
- 時間がかかっている場合は進捗を報告する
